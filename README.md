# GroupBuy 校园拼单 — 产品设计文档

> 碰一碰发起拼单，宿舍/班级秒级组团，自动AA分摊。

## 产品本质

**拼单协调器**，不是外卖平台。不碰菜品、不碰支付、不碰配送。用户在美团/饿了么看好要什么，在 GroupBuy 里协调"谁要什么、一共多少、AA多少"。

---

## 页面结构（5个页面）

```
TabBar
├── 首页 (HomePage)         ← 我的拼单 + 历史复用
├── 我的 (ProfilePage)      ← 统计 + 设置
│
弹出/跳转页面
├── 发起页 (CreatePage)     ← 创建新拼单
├── 详情页 (DetailPage)     ← 拼单详情 + 选品 + AA
└── 加入页 (JoinPage)       ← 碰一碰/扫码后确认加入
```

### 状态机（拼单生命周期）

```
  [拼单中] ──凑够──→ [可下单] ──发起人确认──→ [已下单] ──到货──→ [已到货] ──全员确认──→ [已完成]
     │                                          │
     └──发起人取消──→ [已取消]                    └──超时未到──→ [异常]
     │
     └──超过24h无人加入──→ [已过期]
```

### 每个状态下的页面变化

| 状态 | 进度区域 | 底部按钮 | 可编辑 |
|------|---------|---------|--------|
| **拼单中** | 显示进度条+还差几人/几元 | [邀请更多人] + [凑够了，去下单]（发起人） | 所有人可改自己的选品 |
| **可下单** | "✅ 已达目标！" 绿色提示 | [去下单]（仅发起人） | 仍可改选品 |
| **已下单** | "🚴 已下单，等待送达" | [已到货]（仅发起人） | 锁定，不可改 |
| **已到货** | "📦 外卖到了！快来取！" | [确认收到]（每人） | — |
| **已完成** | "✅ 拼单完成 · 你花了¥10" | [再来一单] | — |
| **已取消** | "❌ 拼单已取消" | [返回首页] | — |

### 交互细节

| 元素 | 操作 | 响应 |
|------|------|------|
| 自己的选品 | 点[修改] | 弹出编辑框，可改名称和价格 |
| 自己的选品 | 点[删除] | 确认弹窗 → 删除 → 重新计算AA |
| [+ 再加一样] | 点击 | 新增一行（文本+价格） |

- 有人改选品 → 费用汇总刷新
- 状态变更 → 全员页面刷新 + 通知

### "累计省了"计算逻辑

```
每单节省 = (原配送费 ÷ 实际人数) × (人数-1)

示例：配送费¥6，4人拼单
  不拼：每人各点，每人付¥6配送 = 总共¥24配送
  拼单：只付1次¥6 = 每人省¥4.5
  本单省了：¥4.5 × 4人 = ¥18

简化版MVP：直接按 (总价÷人数) 与 (单独买的估算价) 的差额
更简化：不算省了多少，只显示"累计拼单XX单，总消费¥XXX"
```

## AA分摊方式

### 默认：按实际消费

```
小明：蛋炒饭¥12 + 加蛋¥2 = ¥14
小红：拌面 = ¥10
小华：炒河粉 = ¥13
总计：¥37

配送费¥6 → 均摊：¥2/人

最终：
小明 ¥14 + ¥2 = ¥16
小红 ¥10 + ¥2 = ¥12
小华 ¥13 + ¥2 = ¥15
```

### 可选：完全均摊

```
总计¥37 + 配送费¥6 = ¥43
¥43 ÷ 3 = ¥14.33/人
```

### 发起人设置

```
┌─ AA方式 ──────────────┐
│ ● 按各自消费 + 均摊杂费  │  ← 默认
│ ○ 完全均摊              │
│ ○ 发起人请客（不AA）     │
└────────────────────────┘

杂费（配送费等）
┌──────┐
│  6   │ 元
└──────┘
```

## 数据模型

```
GroupOrder {
  id: string              // 拼单唯一ID
  category: enum          // 品类：外卖/奶茶/水果/日用/快递/自定义
  title: string           // 店名/商品名
  creatorId: string       // 发起人华为账号ID
  creatorName: string     // 发起人昵称
  targetType: enum        // 目标类型：金额/人数
  targetValue: number     // 目标值
  splitMode: enum         // AA方式：按实际/均摊/请客
  extraFee: number        // 配送费等杂费
  note: string            // 备注
  status: enum            // 拼单中/已下单/已到货/已完成/已取消/已过期
  createdAt: timestamp
  updatedAt: timestamp
}

OrderItem {
  id: string
  orderId: string         // 所属拼单ID
  userId: string          // 华为账号ID
  userName: string
  itemName: string        // 商品名（自由文本）
  price: number           // 价格
  createdAt: timestamp
}

UserSummary {
  userId: string
  totalOrders: number     // 累计拼单数
  totalSpent: number      // 累计消费
}
```

## 首版不做的功能

| 功能 | 原因 | 什么时候做 |
|------|------|-----------|
| 在线支付 | 需要支付牌照，合规风险 | 可能永远不做，用户线下转账 |
| 商家入驻 | 变成平台，复杂度暴增 | V3+ |
| 智能推荐 | 需要数据积累 | V2 |
| 平板公告板 | 非核心路径 | V2 |
| 智慧屏看板 | 非核心路径 | V3 |
| CPS返佣链接 | 需要对接电商平台 | V2 |
